summary: "The rapid adoption of Generative AI has drastically increased software energy consumption, contributing to the \"Red AI\" problem. Currently, developers using third-party LLM APIs lack visibility into their carbon footprint and have no automated way to minimize it. To address this, our project introduces an open-source Python library designed to make LLM usage sustainable. The tool acts as a lightweight wrapper around popular LLM APIs. It calculates the estimated operational and embodied carbon footprint of each request based on token usage and hardware heuristics. Crucially, it features a \"Carbon-Aware Scheduler\" for non-urgent batch processing. By fetching real-time marginal carbon intensity data from the local power grid, the tool automatically delays execution until the grid runs on greener energy. We will validate our solution by running automated NLP benchmarks to quantify the exact CO2eq savings achieved through this time-shifting mechanism. Finally, to ensure high social impact, the library will be published to PyPI, allowing developers to seamlessly integrate sustainable AI practices into their existing workflows with minimal friction."
